# Kotlin Standards

Kotlin conventions for Entur applications. Read [CONVENTIONS.md](../CONVENTIONS.md) first for cross-language standards, and [java.md](java.md) for shared JVM patterns (Spring Boot, testing, dependencies).

## Runtime and Build

- **Kotlin version**: latest stable (currently 2.x)
- **Java target**: 25 (or latest LTS/stable)
- **Build tool**: Gradle with Kotlin DSL (`build.gradle.kts`)
- **Framework**: Spring Boot 3.x with Kotlin support
- **Linting**: Ktlint
- **Dependency management**: Gradle version catalogs (`gradle/libs.versions.toml`)
- **API approach**: Contract-first with OpenAPI Generator (preferred)
- **SQL library**: JetBrains Exposed SQL-DSL (preferred for Kotlin projects)

### build.gradle.kts

Use version catalog aliases for all plugins and dependencies:

```kotlin
plugins {
    alias(libs.plugins.kotlin.jvm)
    alias(libs.plugins.kotlin.spring)
    alias(libs.plugins.spring.boot)
    alias(libs.plugins.spring.dependency.mgmt)
    alias(libs.plugins.openapi.generator)        // For contract-first APIs
}

group = "org.entur"

java {
    toolchain {
        languageVersion = JavaLanguageVersion.of(25)
    }
}

kotlin {
    compilerOptions {
        freeCompilerArgs.add("-Xjsr305=strict")
    }
}

tasks.withType<Test> {
    useJUnitPlatform()
}
```

The `-Xjsr305=strict` flag enables strict null-safety interop with Spring's nullability annotations.

```kotlin
tasks {

            }
            dependencies {
                intoLayer("internal-dependencies") {

                }
                intoLayer("dependencies")
            }
            layerOrder = listOf("dependencies", "internal-dependencies", "spring-boot-loader", "application")
        }
    }
```

## Kotlin-Specific Patterns

### Data Classes for Domain Models

Use `data class` for domain models (not DTOs -- those are generated by OpenAPI):

```kotlin
data class Version(
    val id: Long = 0,
    val netexId: String,
    val created: ZonedDateTime = ZonedDateTime.now(),
    val changed: ZonedDateTime = ZonedDateTime.now(),
    val status: VersionStatus,
    val startDate: LocalDate = LocalDate.now(),
    val endDate: LocalDate? = null,
)
```

### Contract-First REST Controllers

When using OpenAPI Generator, controllers implement generated interfaces:

```kotlin
@RestController
class VersionController(
    private val versionService: VersionService,
    private val versionMapper: VersionMapper,
) : VersionApi {

    @PreAuthorize("hasPermission('product-api-access', 'endre')")
    override fun createVersion(
        createVersionRequest: CreateVersionRequest,
        etClientName: String?,
        xCorrelationId: String?,
    ): ResponseEntity<Version> {
        val domainVersion = versionMapper.toDomain(versionDto)
        versionService.create(domainVersion)
        val createdVersion = versionService.find(createVersionRequest.id)
        return ResponseEntity.status(HttpStatus.CREATED).body(versionMapper.toDto(createdVersion))
    }
}
```

For projects without OpenAPI Generator, use standard Spring annotations:

```kotlin
@RestController
@RequestMapping("/api/v1/routes")
class RouteController(
    private val routeService: RouteService,
) {
    @GetMapping("/{id}")
    fun getRoute(@PathVariable id: String): ResponseEntity<RouteResponse> =
        routeService.findById(id)
            ?.let { ResponseEntity.ok(it) }
            ?: ResponseEntity.notFound().build()

    @PostMapping
    fun createRoute(@Valid @RequestBody request: CreateRouteRequest): ResponseEntity<RouteResponse> {
        val created = routeService.create(request)
        val location = URI.create("/api/v1/routes/${created.id}")
        return ResponseEntity.created(location).body(created)
    }
}
```

### Service Layer

Define service interfaces and implementation classes separately:

```kotlin
// Service interface -- defines the contract and documents behaviour
interface VersionService {
    fun find(netexId: String): Version
    fun create(version: Version)
    fun update(version: Version)
}

// Service implementation -- contains business logic
@Service
class VersionServiceImpl(private val versionDAO: VersionDAO) : VersionService {
    val logger: Logger = LoggerFactory.getLogger(VersionServiceImpl::class.java)

    override fun find(netexId: String): Version {
        versionDAO.find(netexId)?.let { return it }
        throw ElementNotFoundException("Version $netexId not found.")
    }

    override fun create(version: Version) {
        versionDAO.find(version.netexId)?.let {
            throw ElementAlreadyExistException("Version ${it.netexId} exists already.")
        }
        val history = getHistory(version)
        VersionInputValidator.validateNewVersion(version, history)
        versionDAO.create(version)
    }
}
```

### Key Kotlin Principles

- Use `data class` for domain models, value objects, and request/response types
- Use primary constructor injection (not `@Autowired`)
- Use Kotlin null-safety instead of `Optional` -- return `T?` not `Optional<T>`
- Use expression-body functions for simple transformations
- Use trailing commas in parameter lists and collections (improves diffs)
- Use `val` over `var` wherever possible
- Use `sealed class` or `sealed interface` for restricted type hierarchies
- Use `object` for stateless singletons (validators, constants)
- Use `internal` visibility for implementation details not exposed to other modules
- Prefer `when` over `if-else` chains for multi-branch logic
- Use scope functions (`let`, `apply`, `run`, `also`) idiomatically

### Mapper Pattern (DTO to Domain)

Use dedicated `@Component` mapper classes to transform between generated DTOs and domain models. Keep mappers at the API boundary (controller layer):

```kotlin
@Component
class VersionMapper {

    fun toDomain(dto: VersionDto): Version {
        return Version(
            id = 0,                    // Set by database
            netexId = dto.id,
            status = dto.status,
            startDate = dto.startDate,
            endDate = dto.endDate,
            // created/changed NOT set from DTO (read-only fields)
        )
    }

    fun toDto(domain: Version): VersionDto {
        return VersionDto(
            id = domain.netexId,
            status = domain.status,
            startDate = domain.startDate,
            endDate = domain.endDate,
            created = domain.created,
            changed = domain.changed,
        )
    }

    fun toDtoList(domains: List<Version>): List<VersionDto> = domains.map { toDto(it) }
}
```

Alternatively, for simpler projects without OpenAPI Generator, use extension functions:

```kotlin
fun Route.toResponse() = RouteResponse(
    id = id,
    origin = origin,
    destination = destination,
    departureTime = departureTime,
)
```

### Database Access with Exposed SQL-DSL

Exposed is the preferred SQL library for Kotlin projects. It provides a typesafe DSL without ORM magic.

#### Entity Definition

Define database tables as `object` extending `LongIdTable` (or `Table`):

```kotlin
object VersionEntity : LongIdTable("version") {
    val nameOfClass = varchar("name_of_class", VARCHAR_70)
    val created = timestamp("created")
    val changed = timestamp("changed")
    val netexId = varchar("netex_id", VARCHAR_70)
    val status = varchar("status", VARCHAR_70)
    val startDate = date("start_date")
    val endDate = date("end_date").nullable()
}
```

#### DAO Pattern

Use `@Repository`-annotated classes with `transaction { }` blocks:

```kotlin
@Repository
class VersionDAO : BaseDAO<Version> {
    override fun create(element: Version): Long =
        transaction {
            VersionEntity
                .insertAndGetId {
                    it[netexId] = element.netexId
                    it[created] = Instant.now()
                    it[changed] = Instant.now()
                    it[status] = element.status.value
                    it[startDate] = element.startDate
                    it[endDate] = element.endDate
                }.value
        }

    fun find(netexId: String): Version? =
        transaction {
            VersionEntity
                .selectAll()
                .where { VersionEntity.netexId eq netexId }
                .singleOrNull()
                ?.let {
                    Version(
                        id = it[VersionEntity.id].value,
                        netexId = it[VersionEntity.netexId],
                        status = VersionStatus.valueOf(it[VersionEntity.status]),
                        startDate = it[VersionEntity.startDate],
                        endDate = it[VersionEntity.endDate],
                    )
                }
        }
}
```

#### Joins and Queries

Define reusable join functions as extension functions on `Join`:

```kotlin
object VersionEntity : LongIdTable("version") {
    // ... columns ...

    fun Join.joinVersionToPriceableObject() =
        join(
            VersionEntity,
            JoinType.INNER,
            onColumn = netexId,
            otherColumn = PriceableObjectEntity.version,
        )
}
```

### Input Validation

Use dedicated `object` validators for complex business rules:

```kotlin
internal object VersionInputValidator {
    fun validateNewVersion(version: Version, history: Collection<Version>) {
        validateVersionDates(version)
        validateInitialStatus(version)
        validateThatNoDraftOrProposedExistWhenCreatingNewVersion(history)
    }

    private fun validateVersionDates(version: Version) {
        val start = version.startDate
        val end = version.endDate
        if (end != null && !start.isBefore(end)) {
            throw IllegalVersionDatesException("endDate must be strictly after startDate.")
        }
    }
}
```

### Exception Handling

Define typed exceptions in a `common/exception/` package and handle them with `@RestControllerAdvice`:

```kotlin
// Custom exceptions
class ElementNotFoundException(message: String) : RuntimeException(message, null, false, false)
class ElementAlreadyExistException(message: String) : RuntimeException(message)
class IllegalVersionDatesException(message: String) : RuntimeException(message)

// Global exception handler
@RestControllerAdvice
class GlobalExceptionHandler {
    @ExceptionHandler(ElementNotFoundException::class)
    fun handleNotFound(ex: ElementNotFoundException, req: WebRequest) =
        build(HttpStatus.NOT_FOUND, ex, req)

    @ExceptionHandler(IllegalVersionDatesException::class)
    fun handleBadRequest(ex: RuntimeException, req: WebRequest) =
        build(HttpStatus.BAD_REQUEST, ex, req)

    private fun build(status: HttpStatus, ex: Throwable, req: WebRequest): ResponseEntity<ErrorResponse> {
        val body = ErrorResponse(
            error = status.reasonPhrase,
            message = ex.message ?: status.reasonPhrase,
            path = req.getDescription(false).removePrefix("uri="),
        )
        return ResponseEntity.status(status).body(body)
    }
}
```

If using Spring WebFlux with coroutines:

```kotlin

@RequestMapping("/api/v1/routes")
class RouteController(
```

## Testing in Kotlin

### Test Libraries

- **JUnit 5**: Test framework (`@Test`, `@Tag`, `@DisplayName`)
- **Kotest**: Assertions (`shouldBe`, `shouldNotBe`, `shouldHaveSize`, `shouldThrow`)
- **SpringMockK**: Mocking for Spring and Kotlin (`@MockkBean`)
- **TestContainers**: Dockerized PostgreSQL for integration tests
- **Spring Boot Test**: `@SpringBootTest`, `@WebMvcTest`
- **Entur Auth Test**: `TenantJsonWebToken` for simulating authenticated requests

### Unit Tests

Use backtick method names for readable test descriptions. Use Kotest assertions:

```kotlin
class VersionInputValidatorTests {
    @Test
    fun `I should be able to insert a new version with end-date after start-date`() {
        val v = VersionBuilder()
            .withStartDate(LocalDate.now())
            .withEndDate(LocalDate.now().plusDays(2))
            .build()

        assertDoesNotThrow {
            VersionInputValidator.validateNewVersion(v, listOf())
        }
    }

    @Test
    fun `I should not be able to insert a new version with status VERSIONED`() {
        val v = VersionBuilder().withStatus(VersionStatus.VERSIONED).build()

        val ex = assertThrows(IllegalVersionStatusException::class.java) {
            VersionInputValidator.validateNewVersion(v, listOf())
        }
        assertEquals("Invalid version status VERSIONED.", ex.message)
    }
}
```

### Controller Tests (WebMvcTest)

Use `@WebMvcTest` for thin controller tests with mocked services:

```kotlin
@WebMvcTest(VersionController::class)
@Import(VersionMapper::class)
class VersionControllerTests : BaseControllerTest() {

    @MockkBean
    lateinit var versionService: VersionService

    @Test
    fun `When request an existing version get should return 200`(
        @InternalTenant(clientId = "fullAccess") token: String,
    ) {
        every { versionService.find("ENT:Version:1") } returns Version(
            netexId = "ENT:Version:1",
            status = VersionStatus.DRAFT,
        )

        mockMvc.perform(
            get("/v3/versions/ENT:Version:1")
                .header("Authorization", token)
        ).andExpect(status().isOk)
            .andExpect(jsonPath("$.id").value("ENT:Version:1"))
    }
}
```

### Integration Tests (TestContainers)

Use a shared base class for integration tests with TestContainers:

```kotlin
// Test configuration -- reusable PostgreSQL container
@TestConfiguration(proxyBeanMethods = false)
class TestContainersConfig {
    @Bean
    @ServiceConnection
    fun postgresContainer(): PostgreSQLContainer<*> {
        return PostgreSQLContainer(DockerImageName.parse("postgres:17.6-alpine"))
            .withInitScript("test-init.sql")
    }
}

// Base class for all integration tests
@SpringBootTest(webEnvironment = SpringBootTest.WebEnvironment.RANDOM_PORT)
@Import(TestContainersConfig::class)
@ExtendWith(TenantJsonWebToken::class)
@Sql("/test-data/cleanup.sql", executionPhase = Sql.ExecutionPhase.BEFORE_TEST_METHOD)
abstract class BaseServiceTest

// Integration test extending the base class
@DisplayName("VersionService Integration Tests")
class VersionServiceTests : BaseServiceTest() {

    @Autowired
    private lateinit var versionService: VersionService

    @Test
    fun `I should be able to create a new version in DRAFT`() {
        val version = Version(netexId = "ENT:Version:TEST-001", status = VersionStatus.DRAFT)
        versionService.create(version)

        val inserted = versionService.find(version.netexId)
        inserted.netexId shouldBe "ENT:Version:TEST-001"
        inserted.status shouldBe VersionStatus.DRAFT
    }
}
```

### Test Data Builders

Use the builder pattern for constructing test data:

```kotlin
class VersionBuilder {
    private var netexId: String = "ENT:Version:1"
    private var status: VersionStatus = VersionStatus.DRAFT
    private var startDate: LocalDate = LocalDate.now()
    private var endDate: LocalDate? = null

    fun withNetexId(value: String) = apply { netexId = value }
    fun withStatus(value: VersionStatus) = apply { status = value }
    fun withStartDate(value: LocalDate) = apply { startDate = value }
    fun withEndDate(value: LocalDate?) = apply { endDate = value }

    fun build() = Version(netexId = netexId, status = status, startDate = startDate, endDate = endDate)
}
```
